# Fix CPU monitoring stuck at constant value and add missing Azure.Core nuspec dependency

## Problem

CPU-triggered profiling never activates. Debug logs show a constant CPU value (0 on Windows, 6 on Linux) regardless of actual load. Additionally, user-configured `CPUTriggerCooldown` is silently overridden by the 4-hour default.

Separately, the NuGet packages were missing an explicit `Azure.Core` dependency, causing `FileNotFoundException` at runtime when the consuming app resolves a lower version than the profiler was compiled against.

Fixes microsoft/ApplicationInsights-Profiler-AspNetCore#245

## Root Causes

1. **Thread pool starvation blinds CPU monitoring** — `ProcessInfoCPUMetricsProvider` used a fire-and-forget `Task.Delay` loop and `BaselineTracker` used `System.Threading.Timer`. Both depend on thread pool threads. CPU-bound workloads starve the pool, preventing sampling callbacks from executing.
2. **`Math.Floor` truncation** — `BaselineTracker.GetAverage()` truncated CPU values to integers, making small changes invisible (e.g. 0.5% → 0 on Windows, 6.3% → 6 on Linux).
3. **User cooldown not propagated** — `ProfilerSettings` mapped `CPUTriggerThreshold` but not `CPUTriggerCooldown` to `CpuTriggerSettings`, so `PolicyNeedsRefresh()` always overwrote the user's value with the 4-hour default.
4. **Inaccurate initial CPU reading** — A `continue` after the initial snap caused near-zero time delta on the first calculation.
5. **Missing `volatile`** — `_nextValue` was written and read across threads without memory barriers.
6. **Missing `Azure.Core` in nuspec** — `Azure.Identity` only requires `Azure.Core >= 1.27.0`, but the profiler is compiled against `1.47.3`. Without an explicit dependency, NuGet can resolve a lower version.

## Changes

### CPU monitoring fixes (ServiceProfiler submodule)
- Replace async loop in `ProcessInfoCPUMetricsProvider` with a dedicated background thread
- Replace `System.Threading.Timer` in `BaselineTracker` with a dedicated background thread
- Remove `Math.Floor` from `BaselineTracker.GetAverage()`
- Remove `continue` after initial snap to ensure meaningful time delta
- Add `volatile` to `_nextValue` for cross-thread visibility
- Seal `ProcessInfoCPUMetricsProvider`
- Guard CTS disposal on thread exit to avoid `ObjectDisposedException`

### Cooldown propagation fix (main repo)
- Map `CPUTriggerCooldown` → `CpuTriggerSettings.CpuTriggerCooldownInSeconds` in `ProfilerSettings`
- Map `MemoryTriggerCooldown` → `MemoryTriggerSettings.MemoryTriggerCooldownInSeconds` in `ProfilerSettings`

### NuGet packaging fix (main repo)
- Add explicit `Azure.Core` dependency to `ServiceProfiler.EventPipe.Core.nuspec` and `Azure.Monitor.OpenTelemetry.Profiler.nuspec`
- Add `AzureCoreVersion` to `NuspecProperties` in the corresponding csproj/props files

- Add `build-private-package` agent skill for building test NuGet packages
